RUN02 SIMULATION FILES

NAMING CONVENTION AND STORING PLACES

'File naming conventions may seem to be a trivial matter, or an intrusion into one's personal liberty. But try writing file management software for tens of thousands of files without a naming convention', quote Prof. Charles F. Maguire.

Event file

  • OSCAR
    • OSCARv01_RootOut-YYYYYYYYYY-00XX.rootg ====> Oscar event output file in ROOT format
    • OSCARv01_AsciOut-YYYYYYYYYY-00XX.ascii =====> Oscar event output file in ASCII format
  • HIJING
    • HIJING135_Output-YYYYYYYYYY-00XX.zebra ====> Hijing event output file in zebra format

    PISA file
  • From a OSCAR input file
    • PISA2000_OSCAR01-YYYYYYYYYY-00XX.rootg
  • From a HIJING input file
    • PISA2000_HIJv135-YYYYYYYYYY-00XX.rootg

    Here -YYYYYYYYYY is Run Number and 00XX is the Project Number. For example, for project# 1 Run Number starts (or ends!) with -0000001001 and Project Number is 0001.
